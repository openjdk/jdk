{@systemProperty javax.net.debug}

To monitor SSL/TLS activity, you can set the javax.net.debug system property, which determines what trace messages are printed during execution.

If the javax.net.debug property is not defined, debug logging is disabled. If it is defined as empty, the debug logger is System.getLogger("javax.net.ssl"), and applications can customize and configure it using external logging mechanisms. If it is defined and non-empty, a private debug logger that writes to System.err is used.

The following table lists the javax.net.debug options:

SSL/TLS Debug Options
Option Description
all Turn on all debugging
ssl Turn on SSL debugging. If specified by itself, all non-widening filters are enabled. If specified with additional filter options, general SSL debug messages are produced plus just the selected categories. The following filters can be used with ssl:
  • defaultctx: Print default SSL initialization
  • handshake: Print each handshake message. The following sub-option can be used with handshake:
    • verbose: Verbose handshake message printing
  • keymanager: Print key manager tracing
  • record: Enable per-record tracing. The following sub-options can be used with record:
    • packet: Print raw SSL/TLS packets
    • plaintext: Hex dump of record plaintext
  • respmgr: Print OCSP response tracing
  • session: Print session activity
  • sessioncache: Print session cache tracing
  • sslctx: Print SSLContext tracing
  • trustmanager: Print trust manager tracing
expand Use expanded (less compact) output format
help Print help message and exit
